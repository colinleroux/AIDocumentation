# Formal Learning Approach

## AI Application Engineering Development Plan (6-12 Months)

## Objective

To formalize and advance my skills in AI-driven application engineering, with a focus on building secure, scalable, and production-ready internal systems using Python, Flask, Docker, and locally hosted AI models.

This direction aligns with my current work developing internal tools and positions me to deliver higher-value, future-ready solutions for the business.

## Focus Areas

### 1. Retrieval-Augmented AI Systems (RAG)

Goal: Improve the accuracy, reliability, and usefulness of AI-powered internal tools.

Key skills:

- Document preprocessing (chunking, metadata strategies)
- Embeddings and semantic search
- Hybrid retrieval and reranking
- Source attribution and citation
- Retrieval quality evaluation

Outcome:
Development of robust internal knowledge systems that provide accurate, traceable, and context-aware responses.

### 2. Local AI Model Deployment

Goal: Build secure, in-house AI capabilities independent of external services.

Key skills:

- Running and managing local LLMs (for example, Ollama or similar)
- Model selection and performance trade-offs
- Embedding vs generation models
- API-based model serving
- Resource optimization (CPU/GPU, memory, latency)

Outcome:
Deployment of private, cost-controlled AI systems suitable for sensitive or internal data use cases.

### 3. AI System Evaluation and Reliability

Goal: Ensure AI systems are measurable, consistent, and production-ready.

Key skills:

- Creating evaluation datasets
- Measuring answer accuracy and relevance
- Detecting hallucinations and failure cases
- Regression testing across model and prompt changes
- Logging and observability

Outcome:
AI applications that are testable, reliable, and maintainable over time.

### 4. Tool-Integrated AI Workflows

Goal: Extend AI systems beyond Q&A into actionable workflows.

Key skills:

- Designing safe tool interfaces (APIs, scripts, internal services)
- Structured outputs and schema design
- Workflow orchestration (lightweight agents where appropriate)
- Understanding emerging standards (for example, MCP)

Outcome:
AI-enabled tools that can interact with internal systems and automate tasks, not just generate responses.

## Implementation Approach

### Phase 1 (0-3 months)

- Enhance existing RAG system
- Improve retrieval quality and structure
- Add citation and evaluation capabilities
- Begin local model experimentation and benchmarking

### Phase 2 (3-6 months)

- Deploy a stable local model service
- Introduce an evaluation framework for AI outputs
- Optimize performance and reliability of existing tools

### Phase 3 (6-12 months)

- Integrate tool-based workflows into AI systems
- Expand use cases (automation, reporting, internal tooling)
- Refine architecture for scalability and maintainability

## Business Value

- Enables secure, in-house AI solutions (reduced reliance on external APIs)
- Improves internal knowledge access and productivity
- Supports development of custom AI-powered tools tailored to business needs
- Builds capability in a rapidly growing and strategically important area
- Ensures solutions are maintainable, measurable, and production-ready

## Training and Learning Approach

- Targeted online courses in AI engineering (RAG systems, applied LLMs)
- Targeted online courses in backend and API development (advanced Python)
- Hands-on implementation through existing and new internal projects
- Selective study of emerging standards and tools (for example, MCP, local model frameworks)

## Evidence from Current Projects

The development plan is already being applied through active internal project work:

- [MVP Internal RAG](projects/mvp-internal-rag.md): containerized local-model RAG assistant using Flask/FastAPI, Ollama, and Qdrant with citation-backed responses.
- [docx2html Pipeline](projects/docx2html-pipeline.md): production-focused document conversion and content assembly workflow with validation and optional AI-assisted enrichment.

These projects provide practical delivery context for the focus areas above and create a measurable base for the next 6-12 months of capability growth.

## Summary

This plan builds on my existing strengths in Python-based application development and aligns my growth with modern AI engineering practices. It focuses on practical, high-impact skills that support both current responsibilities and future business needs.
